# Writer-Ready Content Brief from Topic

**Purpose:** Produce a brief a writer (human or AI) can execute without asking a single follow-up question — built from SERP overlap, not assumption.

**When to run this:** One per spoke/pillar coming out of `seo-keyword-cluster.md`, immediately before drafting.

**Inputs needed:**
- Target keyword + cluster context (pillar or spoke, and which pillar it belongs to)
- Top 10 current SERP results for the target keyword
- Internal linking targets (homepage, service pages, related spokes already published)

**Process:**
1. Pull the top 10 SERP results and extract: word count range, H2/H3 structure, content format (listicle/guide/comparison/tool), and what each one is missing (the gap you'll fill).
2. Identify the SERP-overlap cluster: which other URLs from the top 10 also rank for 3+ of this term's related queries — that tells you what Google considers "the same topic," and your structure should answer all of it, not just the literal query.
3. Define the **pillar + spokes structure** for the article itself: lead with the direct answer (for AEO/GEO citability, see `seo-geo.md`), then expand into supporting sections matching the question patterns from clustering.
4. Specify required E-E-A-T signals for this specific topic (see `seo-content-audit.md` for the scoring rubric) — named author, cited data source, first-hand specificity.
5. List 3–5 mandatory internal links (anchor text + target URL) and 2–3 external links to primary sources (not competitor blogs).
6. Set the meta title/description character limits and the one sentence the AI Overview should be able to lift verbatim if it cites this page.

**Output format:**
A single brief document: Target keyword | Search intent | Word count target | H2 outline with question-pattern tags | Required internal links | Required E-E-A-T elements | Competitive gap to exploit | One-sentence "citable answer."

**Quality bar:** If the brief doesn't name the specific gap competitors left (not "make it better," but "none of the top 10 include pricing ranges"), send it back — vague briefs produce vague content.
